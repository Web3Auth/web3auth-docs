For making hybrid iOS & Android applications using React Native, we have the
[web3auth-react-native-sdk](https://github.com/Web3Auth/web3auth-react-native-sdk) to integrate Web3Auth into your application.

---

First, install the React Native SDK via npm or yarn.

<Tabs
  defaultValue="npm"
  values={[
    { label: "npm", value: "npm" },
    { label: "yarn", value: "yarn" },
  ]}
>
<TabItem value="npm">

```shell
npm install --save @web3auth/web3auth-react-native-sdk
```

</TabItem>

<TabItem value="yarn">

```shell
yarn add @web3auth/web3auth-react-native-sdk
```

</TabItem>

</Tabs>

After that, depending on whether you are using bare React Native or Expo, you will need to install either the `expo-web-browser` or the
`react-native-web-browser` package to get an implementation of the `WebBrowser` interface.

<Tabs
  defaultValue="expo"
  values={[

>
<TabItem value="expo">

```shell
expo install expo-web-browser
```

</TabItem>

<TabItem value="bare">

```shell
npm install --save @torusresearch/react-native-web-browser
```

</TabItem>

</Tabs>

## Setup instructions

### Expo workflow

Add the your desired url redirect scheme into `app.json` of your Expo project.

### Bare workflow

#### Android

- For Android, the redirectUrl parameter is specificable, and has to be added into the AndroidManifest.xml.

#### iOS

- For iOS, the redirectUrl parameter is also specificable, and does not need to be added as a iOS Custom URL Scheme. However you can still add it to
  the `Info.plist` if you want to do so, but it is not required.
